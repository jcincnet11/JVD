# Internationalization (i18n)

## Adding or Editing Translations

All translatable text lives in two files:

- `messages/en.json` — English (default)
- `messages/es.json` — Spanish

Both files must have identical key structures. If you add a key to one, add it to the other.

### Key Structure

Keys are nested by section:

```json
{
  "nav": { "services": "Services", ... },
  "hero": { "headline": "...", "subheadline": "...", ... },
  "services": { "heading": "...", "card1": { "title": "...", ... }, ... },
  ...
}
```

### Using Translations in Components

```tsx
// Server or client component
import { useTranslations } from 'next-intl';

export default function MyComponent() {
  const t = useTranslations('sectionName');
  return <h2>{t('heading')}</h2>;
}
```

For raw array access (e.g., feature lists):

```tsx
const items = t.raw('bundles.launch.covered') as string[];
```

## Locale Routing

- `/en` — English (default)
- `/es` — Spanish
- `/` — Redirected to `/en` by middleware

The locale is determined by the `[locale]` URL segment. The language toggle in Nav links directly to `/{otherLocale}`.

## Adding a New Locale

1. Create `messages/{locale}.json` with all keys translated
2. Add the locale to the `locales` array in `middleware.ts`
3. Add the locale to the `locales` array in `i18n/request.ts`
4. Add the locale to the `locales` array in `app/[locale]/layout.tsx`
5. Update the language toggle in `components/Nav.tsx` to handle the new option
